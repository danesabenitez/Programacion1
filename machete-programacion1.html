📖 Unidad 1 — Conceptos Fundamentales
Definiciones clave
Programación: Elaboración de programas para la resolución de problemas mediante computadoras.
Algoritmo: Secuencia finita, ordenada y no ambigua de instrucciones para resolver un problema.
Programa: Colección de instrucciones que, al ser ejecutadas por el CPU, llevan a cabo una tarea.
Lenguaje de Programación: Conjunto de reglas y convenciones para escribir programas que la computadora puede ejecutar.
Sintaxis: CÓMO se escribe (reglas de estructura). — Semántica: QUÉ hace (significado). Un prog. sintácticamente correcto puede ser semánticamente incorrecto.

Prueba de Escritorio

Verificación manual y lógica de un algoritmo sin ejecutarlo en computadora. Permite detectar:

  • Pasos fuera de orden
  • Pasos faltantes o de más
  • Bucles que se repiten más o menos veces de lo debido
Pasos: identificar código → analizar lógica → elegir valores de entrada → calcular manualmente → comparar resultados esperados vs reales → corregir → repetir con otros valores.

🗂 Unidad 2 — Estructuras de Datos
Definición
Estructura de datos: Forma de organizar y almacenar datos de manera que puedan ser utilizados eficientemente. Permite diseñar la solución correcta para un determinado problema.
Clasificación general
Estáticas: El tamaño en memoria se define ANTES de ejecutar el programa y NO puede modificarse durante la ejecución.
  • Primitivas: Entero, Real, Booleano, Carácter
  • Compuestas: Arreglos, Strings, Archivos, Registros
Dinámicas: El tamaño en memoria PUEDE modificarse durante la ejecución del programa.
  • Lineales: Listas, Pilas, Colas
  • No lineales: Árboles, Grafos

Arreglos
Arreglo: Colección finita, homogénea y ordenada de elementos del mismo tipo. Es una estructura estática compuesta.

Finita = tiene tamaño definido | Homogénea = todos del mismo tipo | Ordenada = cada elemento tiene posición (índice)

Strings (Cadenas)
String: Secuencia ordenada de longitud finita de caracteres. Puede contener letras (a-z, A-Z), números (0-9), espacios y símbolos (!,@,%…)
Archivo o Fichero
Archivo: Colección ordenada de datos relacionados que se almacenan de forma permanente en memoria no volátil (disco). A diferencia de la RAM, los datos sobreviven al apagado del equipo.
Estructuras Dinámicas Lineales

Cada elemento tiene un único sucesor y un único predecesor. Sus elementos están ubicados uno al lado del otro en forma lineal.

PILA (Stack) — LIFO
Last In, First Out = último en entrar, primero en salir.

Operaciones: push (apilar) y pop (desapilar). Se trabaja siempre por el TOPE.
COLA (Queue) — FIFO
First In, First Out = primero en entrar, primero en salir.

Operaciones: encolar y desencolar. Entra por el FINAL, sale por la CABECERA.
LISTA ENLAZADA: Cada elemento (nodo) contiene un dato y un enlace a la posición del siguiente elemento. Teniendo la referencia del inicio se accede a todos los elementos.
Estructuras Dinámicas NO Lineales

Cada elemento puede estar enlazado a CUALQUIER otro componente (varios sucesores o varios predecesores).

ÁRBOL: Estructura jerárquica. Cada elemento tiene un único ANTECESOR y puede tener varios SUCESORES.

Árbol binario: máximo 2 nodos por elemento.
GRAFO: Estructura matemática con nodos (vértices) y líneas que los unen (aristas/arcos). Sin restricciones de jerarquía.

🔀 Unidad 3 — Sentencias de Control
Programación Estructurada — Teorema de Dijkstra

Todo programa puede escribirse usando únicamente 3 estructuras de control:

Secuencia: instrucciones una tras otra, sin bifurcaciones.
Selección: una u otra rama según condición. → if / switch
Iteración: repite instrucciones mientras la condición se cumpla. → while / for / do-while
IF Simple

Ejecuta instrucciones solo si la condición es verdadera. Si es falsa, el bloque se omite.

if (condicion)
{
    sentencia1;
    sentencia2;
}
Sin llaves, solo la PRIMERA instrucción pertenece al if.
IF–ELSE (doble)

Siempre ejecuta una de las dos ramas: la del if (verdadero) o la del else (falso).

if (condicion)
{
    /* verdadero */
    sentencia1;
}
else
{
    /* falso */
    sentencia2;
}
IF Anidado (else if)

Múltiples alternativas encadenadas.

if (condicion1)
    instruccion1;
else if (condicion2)
    instruccion2;
else if (condicion3)
    instruccion3;
else
    instruccion4;   // ninguna de las anteriores
Ejemplo saludo: if(hora<12) "Buenos días" else if(hora<18) "Buenas tardes" else "Buenas noches"
SWITCH — Selección múltiple

Compara el valor de una variable contra distintas constantes. Útil para reemplazar IF–ELSE IF largo.

switch (selector)
{
    case 1:
        instrucciones;
        break;
    case 2:
        instrucciones;
        break;
    default:
        instrucciones;
        break;
}
ReglaDetalle
Tipo del selectorSolo int o char. NO float ni double.
Valores de caseDeben ser CONSTANTES (no variables)
breakOBLIGATORIO al final de cada case. Sin él, cae al siguiente case ("fall-through")
defaultOpcional. Se ejecuta cuando ningún case coincide.
ComparaciónSolo igualdad. Para rangos (>, <) usar IF.
Para char usar comillas simples: case 'a':
FOR — Repeticiones conocidas

Usar cuando se sabe de antemano cuántas veces repetir. Automatiza inicialización + condición + actualización.

for (inicializacion ; condicion ; actualizacion)
{
    instruccion1;
    instruccion2;
}
ParteCuándo se ejecutaEjemplo
InicializaciónUNA SOLA VEZ antes del buclei = 0
CondiciónANTES de cada iteracióni < 10
ActualizaciónDESPUÉS de cada iteracióni++
// Imprimir 0 al 9
for (i = 0; i < 10; i++)
    printf("%d\n", i);
// Al salir, i vale 10 (condición de salida)
// Números pares de 50 a 20 (decremento)
for (i = 50; i >= 20; i -= 2)
    printf("%d\n", i);
WHILE — Condición desconocida

Evalúa condición ANTES de ejecutar. Si es falsa desde el principio, el cuerpo NUNCA se ejecuta.

while (condicion)
{
    instruccion1;
    actualizacion_contador;  // ¡no olvidar!
}
La variable de control debe: (1) inicializarse ANTES del while, (2) comprobarse en la condición, (3) actualizarse DENTRO. Si se olvida actualizar → bucle INFINITO.
// Validar entrada: solo acepta valor positivo
int a = 0;
while (a <= 0)
{
    printf("Ingrese valor positivo: ");
    scanf("%d", &a);
}
DO-WHILE — Mínimo 1 ejecución

Evalúa condición DESPUÉS. El cuerpo se ejecuta siempre al menos una vez.

do
{
    instruccion1;
    actualizacion;
} while (condicion);   // ← punto y coma OBLIGATORIO
El punto y coma final en while(cond); es OBLIGATORIO. Es error muy común olvidarlo.
// Validar con do-while (menús, validaciones)
int a;
do
{
    printf("Ingrese valor positivo: ");
    scanf("%d", &a);
} while (a <= 0);
Contador vs Acumulador
Contador: se incrementa en cantidad CONSTANTE.
contador++; o contador = contador + 1;
Acumulador: se incrementa en cantidad VARIABLE (depende de un dato).
suma = suma + numero;
Todo contador/acumulador debe inicializarse en 0 ANTES del bucle y actualizarse DENTRO.

++i vs i++ (importante)
++i (pre-incremento): el valor se modifica ANTES de usarlo en la expresión.
i++ (post-incremento): el valor se modifica DESPUÉS de usarlo en la expresión.
int i = 5;
printf("%d", ++i);  // imprime 6, i queda en 6
printf("%d", i++);  // imprime 6, i queda en 7
